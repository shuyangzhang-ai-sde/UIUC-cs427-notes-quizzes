Design Patterns
Creational
How objects are created
Singleton
One instance · global access
Factory Method
Subclasses choose which class to instantiate
Abstract Factory
Interface for families of related objects
Builder
Step-by-step construction of complex objects
Prototype
Clone an existing object to make a new one
Structural
How classes and objects are composed
Composite KEY
Tree structure · treat individuals and groups uniformly
Three roles
Component interface Leaf Composite container
⚠ Child-management methods (add/remove) belong on the Composite class, not on the Component interface
Adapter
Adapt interfaces · bridge incompatible APIs
Decorator
Attach responsibilities to objects dynamically
Facade
One simple high-level interface to a subsystem
Behavioral
Object collaboration and responsibility
Observer KEY
State change → notify all subscribers; event notification across listeners
Strategy KEY
Family of algorithms · interchangeable at runtime
Interpreter KEY
Parse grammars · build an AST
Interpreter roles
Terminal (leaf) NonTerminal (internal) Context carries state
⚠ The AST is built from Expression instances; Context is not a tree node
Iterator
Sequential access over a collection
Visitor
Add new operations without changing class structure
State
Behavior changes with state
Key exam points
Point 01
A class can play multiple patterns
A class can participate in more than one design pattern
Point 02
Do not mix up the three families
Creational · Structural · Behavioral
Point 03
Strategy vs Visitor
Strategy swaps the whole algorithm; Visitor adds operations without changing classes

→ Strategy Pattern

→ Observer Pattern
Parse a custom language or expression grammar
→ Interpreter Pattern

→ Iterator Pattern
