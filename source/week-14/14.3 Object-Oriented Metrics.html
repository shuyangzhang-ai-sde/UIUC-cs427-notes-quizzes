14.3 — Object-Oriented Metrics

Chapter 14 OO Metrics Software Engineering
Introduction
Object-Oriented (OO) Metrics
OO Metrics Complexity Maintainability Testability Reusability
OO Metrics are quantitative measures used to evaluate the structural properties of object-oriented software. They help assess complexity, coupling, and inheritance depth — all of which directly impact maintainability, testability, and reusability.
WMC
Weighted Methods Per Class
Sum of method complexities in a class
DIT
Depth of Inheritance Tree
How deep a class is in the hierarchy
CBO
Coupling Between Object Classes
How tightly classes depend on each other
NOC
Number of Children
Direct subclasses of a class
Core Principle
Generally, lower is better for WMC, DIT, and CBO — lower values indicate simpler, less coupled, more testable classes. NOC is more nuanced.
Metric
WMC — Weighted Methods Per Class
WMC Method Complexity Cyclomatic Complexity Class Size
WMC measures the sum of complexities of all methods defined in a class. If all methods are treated as equally complex (weight = 1), WMC simply equals the number of methods in the class.
AspectDetail
What it measuresSum of individual method complexities (e.g., cyclomatic complexity) across all methods in a class
FormulaWMC = Σ complexity(method_i) for all methods in class
Uniform weightIf each method has weight 1, WMC = total number of methods
High WMC impactHarder to understand, test, maintain, and reuse the class
Low WMC impactSimpler class; more likely to be reusable and testable
  • A class with many complex methods is harder to understand and test
  • High WMC suggests the class may be doing too much — violating the Single Responsibility Principle
  • WMC is one of the Chidamber & Kemerer (CK) metrics suite
Exam Point
WMC is the metric that measures the sum of complexities of methods in a class — not DIT, CBO, or NOC.
Metric
DIT — Depth of Inheritance Tree
DIT Inheritance Class Hierarchy Reuse Predictability
DIT measures how deep a class is in the inheritance hierarchy — i.e., the maximum number of ancestor classes from the class to the root.
DIT ValueImplication
Low DITFewer inherited methods; class behavior is easier to predict and understand
High DITMore inherited methods → more reuse, but behavior is harder to predict and class is more complex
  • Reuse ↑ as DIT increases — the class inherits more methods from ancestors
  • Predictability ↓ as DIT increases — harder to trace which inherited method is being called
  • DIT applies to both single and multiple inheritance hierarchies
  • DIT is related to class complexity — deeper hierarchies introduce more indirect dependencies
Common Misconception
Higher DIT does NOT make behavior easier to predict — it's the opposite. More inherited methods = more complexity and harder-to-predict behavior.
Exam Point
Higher DIT → more reuse but harder to predict + more complex. DIT is not limited to single inheritance.
Metric
CBO — Coupling Between Object Classes
CBO Coupling Dependencies Testability Reusability
CBO counts the number of other classes a class is coupled to — i.e., how many distinct classes it depends on or that depend on it (excluding inheritance relationships).
CBO LevelEffect on TestabilityEffect on Reusability
Low CBOEasier to testMore reusable
High CBOHarder to testLess reusable
  • High CBO means the class depends on many others → changes elsewhere can break it
  • Tightly coupled classes require more setup and mocking during testing
  • A highly coupled class is harder to reuse in a different context — it carries many dependencies
  • Goal: low coupling, high cohesion is a fundamental OO design principle
Exam Point — False Statement Alert
"Higher CBO makes a class easier to test and more reusable" is FALSE. Higher CBO does the opposite: it makes testing harder and reuse more difficult.
Metric
NOC — Number of Children
NOC Subclasses Inheritance Influence
NOC counts the number of immediate subclasses (direct children) of a class in the class hierarchy.
NOC LevelImplication
High NOCClass is highly reused via inheritance; changes to it impact many subclasses — more testing needed
Low NOCLimited reuse through inheritance; less ripple effect from changes
  • A high NOC class has greater influence over the system — it must be designed carefully
  • More children = more thorough testing required for the parent class
  • NOC indicates the breadth of reuse (whereas DIT indicates depth)
  • NOC does not directly measure complexity like WMC — it measures inheritance fan-out
DIT vs. NOC
DIT = how deep a class is (ancestors above it). NOC = how wide its influence is (children below it). Both are part of the CK metrics suite.
Summary
All Four OO Metrics at a Glance
WMC DIT CBO NOC
MetricFull NameMeasuresHigh Value Means
WMC Weighted Methods Per Class Sum of method complexities in a class More complex, harder to test & maintain

Design Goal
  • Low WMC → simpler methods, easier to test and understand
  • Low DIT → more predictable behavior, lower complexity
  • Low CBO → less coupling, more testable and reusable
  • NOC → high NOC requires careful design of the parent class
One-Line Summary
WMC = method complexity sum · DIT = inheritance depth · CBO = coupling count · NOC = direct children count. All four are part of the Chidamber & Kemerer (CK) Metrics Suite.
