Parameterized Test
Slide content
  • Must declare at least one source that provides arguments for each invocation of the parameterized test, e.g.,
    • @ValueSource
      • Argument of short, byte, int, long, float, double, char, java.lang.String, or java.lang.Class
@ParameterizedTest @ValueSource(strings = { "racecar", "radar", "able was I ere I saw elba" }) void palindromes(String candidate) { assertTrue(isPalindrome(candidate)); }

ConsoleLauncher will print output as below

palindromes(String) ✔ ├─ [1] racecar ✔ ├─ [2] radar ✔ └─ [3] able was I ere I saw elba ✔

(Bechtold et al., 2018)

Transcript

A parameterized test is a unit test with parameters, unlike conventional unit tests which do not have parameters for the unit test methods. In this example, we use ValueSource as a way for providing arguments of specific types such as a short byte integer, long flow the double, characters, string or generally class. In our example, we supply three string values one-by-one for us to invoke the parameters tests. After we use the execution engine of the unit test to run such parameter tests which the supplied source of the argument values, we can have three concrete conventional unit tests being produced, with each of them corresponding to the argument values being listed from the ValueSource. Besides using the ValueSource to supply argument values for those premium type argument values for the parameter test, we can also use EnumSource for the arguments of enumerate type of the parameters tests.

Other Sources of Arguments: EnumSource
Slide content
  • @EnumSource
    • Argument of Enum Type
@ParameterizedTest @EnumSource(value = TimeUnit.class, names = { "DAYS", "HOURS" }) void testWithEnumSourceInclude(TimeUnit timeUnit) { assertTrue(EnumSet.of(TimeUnit.DAYS, TimeUnit.HOURS).contains(timeUnit)); }

(Bechtold et al., 2018)

Transcript

For our example, we have the enumerate type TimeUnit for the parameter test, then we can specify the argument values using the EnumSource with the value as TimeUnit.class, and then using the concrete names of the constants being defined on the TimeUnit class as days and hours. Then when we execute these parameter test with the supplied argument values. We parse in two different argument values with the enumerate type constant.

Other Sources of Arguments: MethodSource
Slide content
  • @MethodSource
    • One or more factory methods of the test class or external classes
@ParameterizedTest @MethodSource("stringProvider") void testWithSimpleMethodSource(String argument) { assertNotNull(argument); } @ParameterizedTest static Stream<String> stringProvider() { return Stream.of("foo", "bar"); }

Streams for primitive types (DoubleStream, IntStream, and LongStream)

  • @MethodSource: multiple parameters
@ParameterizedTest @MethodSource("stringIntAndListProvider") void testWithMultiArgMethodSource(String str, int num, List<String> list) { assertEquals(3, str.length()); assertTrue(num >=1 && num <=2); assertEquals(2, list.size()); } static Stream<Arguments> stringIntAndListProvider() { return Stream.of( arguments("foo", 1, Arrays.asList("a", "b")), arguments("bar", 2, Arrays.asList("x", "y")) ); }
  • @MethodSource: external, static factory method (fully qualified method name)
package example; import java.util.stream.Stream; import org.junit.jupiter.params.ParameterizedTest; import org.junit.jupiter.params.provider.MethodSource; class ExternalMethodSourceDemo { @ParameterizedTest @MethodSource("example.StringsProviders#tinyStrings") void testWithExternalMethodSource(String tinyString) { // test with tiny string } } class StringsProviders { static Stream<String> tinyStrings() { return Stream.of(".", "oo", "OOO"); } }

(Bechtold et al., 2018)

Transcript

We can also use MethodSource, to specify one or more factory methods, of the test class or an external class, in order to supply argument values for the parameter tests. For our example, we specify stringProvider as the name of the factory method that we are going to use. Then the execution engine we search the test class for search tests factory method matching the specified string of the method name. These factory method return stream of two string values.

By default the stream would indicate the lease of the strings, such as a foo and bar. We can also define some other stream types for other primitive types such as a double, integer and long, as a return type of the factory methods that we are going to specify for the parameter tests. So far we have shown the examples of parameters test with a single parameter. Of course, we can also specify multiple parameters for a parameters test. In such situations, we can use return type for the factory measure a stream of arguments. For our example, here we specify the three argument values for the three parameters of the parameters tests, and then we use arguments to encapsulate all these three elements together as one of the elements being returned in the stream, being fed as argument values for the parameters tests. Using the MethodSource to specify the factory method, we can also put down fully qualified method name such as the patch name, class name, and the method name for the factory methods being defined as external static factory method.

Other Sources of Arguments: CsvSource
Slide content
  • @CsvSource: comma-separated values (i.e., String literals)
@ParameterizedTest @CsvSource({ "foo, 1", "bar, 2", "'baz, qux', 3" }) void testWithCsvSource(String first, int second) { assertNotNull(first); assertNotEquals(0, second); }

(Bechtold et al., 2018)

Transcript

We can also use CsvSource to specify a list of string literals separated by commas. As example here, for the two parameters of the parameter test, one string the other as integer, we can specify three pairs of the values for us to specify with CsvSource. So that we have these implicit conversion from the string values as one, two, three, being converted to the integer type during the execution of the parameterized test with such argument values. Note that, here we use a single quotation mark to indicate the quote characters in stream.

Other Sources of Arguments: CsvFileSource
Slide content
  • @CsvFileSource: CSV files from the classpath (each line results in one invocation of the parameterized test)
@ParameterizedTest @CsvFileSource(resources = "/two-column.csv", numLinesToSkip = 1) void testWithCsvFileSource(String first, int second) { assertNotNull(first); assertNotEquals(0, second); }

two-column.csv

Country, reference Sweden, 1 Poland, 2 "United States of America", 3

Using a double quote " as the quote character

(Bechtold et al., 2018)

Transcript

We can also use CsvFileSource to specify the name of the csv file from the class path. Here is example of the csv file with the name as a two column. Then we can specify CsvFileSource with the resources being the file name. In addition, we also specify the number lines to skip as one, because for our csv file the first line indicates the header of the data in the csv file. We should skip this first row or first line, when we read the lines from the csv file as one line being one list of the arguments being fed to the parameters of the parameters test. As you can see, we have three lines, starting from the second line, each line of these three lines will be used as the argument values for invoking this parameters tests. Note that, here within the csv file, we use double quotation mark to indicate the quote characters.

Other Sources of Arguments: ArgumentsSource
Slide content
  • @ArgumentsSource: a custom, reusable ArgumentsProvider (declared as top-level class or as static nested class)
@ParameterizedTest @ArgumentsSource(MyArgumentsProvider.class) void testWithArgumentsSource(String argument) { assertNotNull(argument); } public class MyArgumentsProvider implements ArgumentsProvider { @Override public Stream<? extends Arguments> provideArguments(ExtensionContext context) { return Stream.of("foo", "bar").map(Arguments::of); } }

(Bechtold et al., 2018)

Transcript

You can also use ArgumentsSource to specify a custom reducible arguments provider class, being declared as a top level class or as a static nested class. For our example, we declare MyArgumentsProvider as the class name to supply factory methods for being used with these parameterized test. Then the execution engine, will look for applicable public method defined on these MyArgumentsProvider, with the return type as applicable on the parameters of the parameters test.
