Parameterized Test
Slide content
  • Must declare at least one source that provides arguments for each invocation of the parameterized test, e.g.,
    • @ValueSource
      • Argument of short, byte, int, long, float, double, char, java.lang.String, or java.lang.Class
@ParameterizedTest @ValueSource(strings = { "racecar", "radar", "able was I ere I saw elba" }) void palindromes(String candidate) { assertTrue(isPalindrome(candidate)); }

(Bechtold et al., 2018)

Transcript

When we use value source or other annotations to indicate the source of the arguments to be string literals at runtime, when we use the execution engine to run the parameterized test, we carry out implicit conversion from such string literals, to the target parameter type. For example, if our target parameter type is Boolean, then we convert string value true to be the Boolean value true.

Other Sources of Arguments: EnumSource
Slide content
  • @EnumSource
    • Argument of Enum Type
@ParameterizedTest @EnumSource (value = TimeUnit.class, names = { "DAYS", "HOURS" }) void testWithEnumSourceInclude(TimeUnit timeUnit) { assertTrue(EnumSet.of(TimeUnit.DAYS, TimeUnit.HOURS).contains(timeUnit)); }

(Bechtold et al., 2018)

Transcript

If the type is character, then we convert a single characters in the string literals to be the characters that we want. Similarly for integers, we convert the integer string to the integer values, and for Enumeric subclass we convert the string for indicating the constants defined on the Enumeric type such as a TimeUNIT.SECONDS converted from seconds string value. For java.io.File, we can convert the string values indicating the file name into a file object by putting the string value as argument value for the constructor or for file.

Other Sources of Arguments: MethodSource
Slide content
  • @MethodSource
    • One or more factory methods of the test class or external classes
@ParameterizedTest @MethodSource("stringProvider") void testWithSimpleMethodSource(String argument) { assertNotNull(argument); } @ParameterizedTest static Stream<String> stringProvider() { return Stream.of("foo", "bar"); }

Streams for primitive types [DoubleStream, IntStream, and LongStream]

(Bechtold et al., 2018)

Transcript

We can also carry out Fallback string-to-object conversion, by converting string literals to the target parameterized type by using such mechanism. As our example, we use a value source to specify a string "42 cats" as the source for the argument values. Given that our target parameter type is book then, our execution engine we'll look for applicable factory methods being defined on our target parameter type book. If you look for a non private method being defined on the target parameter type and such method must be static.

Other Sources of Arguments: MethodSource Cont.
Slide content
  • @MethodSource: multiple parameters
@ParameterizedTest @MethodSource("stringIntAndListProvider") void testWithMultiArgMethodSource(String str, int num, List<String> list) { assertEquals(3, str.length()); assertTrue(num >= 1 && num <= 2); assertEquals(2, list.size()); } static Stream<Arguments> stringIntAndListProvider() { return Stream.of( arguments("foo", 1, Arrays.asList("a", "b")), arguments("bar", 2, Arrays.asList("x", "y")) ); }

(Bechtold et al., 2018)

Transcript

These applicable factory methods must have a single string argument and the return of such applicable factory methods must be the target parameter type. In addition, the execution engine will also look for non-private constructor from the target parameterized type with a single string argument. If both the applicable factory constructor and the applicable factory method can be found on the target parameter type, then the factory method will be selected, not the applicable factory constructor. If multiple applicable factory methods are found on the target parameter type, then none of these factory methods will be applied.

Other Sources of Arguments: MethodSource Cont.
Slide content
  • @MethodSource: external, static factory method (fully qualified method name)
package example; import java.util.stream.Stream; import org.junit.jupiter.params.ParameterizedTest; import org.junit.jupiter.params.provider.MethodSource; class ExternalMethodSourceDemo { @ParameterizedTest @MethodSource("example.StringsProviders#tinyStrings") void testWithExternalMethodSource(String tinyString) { // test with tiny string } } class StringsProviders { static Stream<String> tinyStrings() { return Stream.of(".", "oo", "000"); } }

(Bechtold et al., 2018)

Transcript

We can also carry out explicit conversion by annotating the parameter of the parameterized test with such annotation indicating the class that we are going to use to find out the conversion measures for converting from the source to the target parameter type. As you can see here, we can find out two string argument converters as the specified converter class. Then we look for such converter method for converting the source as specified with the source of the argument. Then, we convert the object to the target type as the argument here as a string type. Then, the return value of such a conversion method will be used as the argument values.

Other Sources of Arguments: CsvSource
Slide content
  • @CsvSource: comma-separated values (i.e., String literals)
@ParameterizedTest @CsvSource({ "foo, 1", "bar, 2", "'baz, aux', 3" }) void testWithCsvSource(String first, int second) { assertNotNull(first); assertNotEquals(0, second); }

(Bechtold et al., 2018)

Transcript

We can also specify our parameters tests with parameter type argument assessor. By using these carefully designed argument type, we can invoke getString or get to allow us to convert individual string values being specified in the argument source to our target argument types being used within the body of the parameterized tests for us to construct the needed object type such as a person in this example.

Other Sources of Arguments: CsvFileSource
Slide content
  • @CsvFileSource: CSV files from the classpath (each line results in one invocation of the parameterized test)
@ParameterizedTest @CsvFileSource(resources = "/two-column.csv", numLinesToSkip = 1) void testWithCsvFileSource(String first, int second) { assertNotNull(first); assertNotEquals(0, second); }

two-column.csv

Country, reference Sweden, 1 Poland, 2 "United States of America", 3

Using a double quote " as the quote character

(Bechtold et al., 2018)

Transcript

For each of the four values of the argument source, we get the first one being string and the second one also strings. But for the third argument, we convert it to the gender class and the first one converted to local date. These four argument types are the needed argument types for the constructor of person class.

Other Sources of Arguments: ArgumentsSource
Slide content
  • @ArgumentsSource: a custom, reusable ArgumentsProvider (declared as top-level class or as static nested class)
@ParameterizedTest @ArgumentsSource(MyArgumentsProvider.class) void testWithArgumentsSource(String argument) { assertNotNull(argument); } public class MyArgumentsProvider implements ArgumentsProvider { @Override public Stream<? extends Arguments> provideArguments(ExtensionContext context) { return Stream.of("foo", "bar").map(Arguments::of); } }

(Bechtold et al., 2018)

Transcript

Instead of constructing the objects that will be needed in order to test different behaviors within the parameterized test, we can specify the construction of the target objects outside of the parameter test. But instead, we still parse in the target argument type as a person for the example we have seen. However, we annotate these argument types with @AggregateWith as the specific class name that we are going to look for the conversion or aggregation of the individual string values into the target object type as a person.

In particular, we see that the person aggregator being defined as a public class and then we look for non private methods for returning the target type person and with argument assessor type and the parameter contexts for retrieving the individual argument values from the string literals being listed in the argument source.
